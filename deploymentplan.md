# Deployment Plan: Mutual Fund FAQ Assistant

This document provides a step-by-step guide to deploy the **Mutual Fund FAQ Assistant** using **Railway** (for the FastAPI backend & database) and **Vercel** (for the static React frontend).

---

## 🛠️ Architecture Summary

1. **Backend (FastAPI)**: Runs python processes for API endpoints, web scraping, and database operations. Must be deployed to a platform that supports persistent services (e.g., Railway).
2. **Databases (SQLite & ChromaDB)**: Created dynamically on system startup by `src/data/scheduler.py` via `run_ingestion()`.
3. **Frontend (React)**: A single-page application (`src/app/index.html`) using CDN-loaded React and Tailwind. It can be served directly from FastAPI (All-in-One) or hosted on Vercel.

---

## 📋 Deployment Option A: All-in-One on Railway (Recommended)

In this setup, Railway hosts both the FastAPI backend and serves the frontend as static files. This keeps the configuration simple and requires zero changes to the frontend code.

### 1. Prepare Configuration Files
Make sure the following files are in the root of your project:
- **`requirements.txt`**: Ensure all python packages are listed (FastAPI, Uvicorn, BeautifulSoup4, ChromaDB, Sentence-Transformers, etc.).
- **`Procfile`**: Create a file named `Procfile` in the root directory containing:
  ```text
  web: uvicorn src.app.main:app --host 0.0.0.0 --port $PORT
  ```

### 2. Set Up Railway
1. Go to [Railway](https://railway.app) and log in.
2. Click **New Project** -> **Deploy from GitHub repo**.
3. Select your repository `Mutual-Fund-Assistant`.
4. Click **Add Variables** to configure the environment:
   * `GROQ_API_KEY`: *Your Groq API Key*
   * `GEMINI_API_KEY`: *Your Gemini API Key*
   * `LLM_PROVIDER`: `groq`
   * `LLM_MODEL`: `llama-3.3-70b-versatile`
5. Go to **Settings** under your service and click **Generate Domain** under the **Networking** section.
6. Railway will build and deploy the application automatically. The database will be initialized upon startup.

## 📋 Deployment Option B: Split Deployment (Vercel + Railway)

If you prefer to host the static React frontend on Vercel for faster global CDN delivery and the FastAPI backend on Railway, follow this setup.

### Step 1: Update Frontend URL
Open [src/app/index.html](file:///Users/aarushigrover/Desktop/Mutual%20Fund%20FAQ%20Assistant/src/app/index.html) and configure the production backend URL:
```javascript
// Line 150: Update the empty string with your Railway service URL
const BACKEND_URL = 'https://your-railway-app-url.up.railway.app';
```
*Note: Make sure to commit and push this change to GitHub.*

### Step 2: Deploy Backend to Railway
Follow the **All-in-One** setup steps above to deploy the FastAPI backend to Railway. Note the public URL generated by Railway.

### Step 3: Deploy Frontend to Vercel
Vercel will host the static `index.html` file.
1. Create a `vercel.json` file in the root directory to map requests correctly:
   ```json
   {
     "rewrites": [
       {
         "source": "/(.*)",
         "destination": "/src/app/index.html"
       }
     ]
   }
   ```
2. Log into [Vercel](https://vercel.com).
3. Click **Add New** -> **Project** -> Select `Mutual-Fund-Assistant` repository.
4. In **Build & Development Settings**:
   * **Framework Preset**: Other
   * **Output Directory**: `src/app` (since your single index.html file is inside `src/app`)
5. Click **Deploy**. Vercel will serve your index.html globally.

## 🔒 Security & CORS Configuration
If you choose the Split Deployment (Option B), ensure your backend allows requests from your Vercel domain. In [src/app/main.py](file:///Users/aarushigrover/Desktop/Mutual%20Fund%20FAQ%20Assistant/src/app/main.py):
```python
# Enable CORS for production Vercel domain
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://your-vercel-domain.vercel.app"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```
